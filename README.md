# agent-kefu

`agent-kefu` 是一个面向公开学习与拆解的 AI 客服项目整理版仓库。

这个仓库保留了一个“平台接入层 + Prompt 路由 + 会话上下文 + 商品知识库”的完整项目骨架，但已经移除了不适合公开发布的敏感信息、真实业务数据和私有部署依赖，更适合拿来学习架构思路、阅读项目拆解文档，或作为你自己的客服 Agent 原型起点。

## 项目定位

- 这是学习版与公开版仓库，不是可直接上线的生产交付包
- 仓库重点在于展示 AI 客服项目如何组织代码、知识库、提示词和状态管理
- 公开版默认不应连接真实平台账号、Cookie 或私有接口

## 你能在这里看到什么

- 一个客服 Agent 项目的基础代码入口与目录组织方式
- 按场景拆分的 Prompt 模板
- 商品资料、FAQ、教程、标准回复等知识库模板
- 面向公开阅读重新整理的项目解析文档与验证记录

## 目录结构

```text
agent-kefu/
├─ main.py                     # 项目主入口
├─ prompts/                    # 分类、咨询、定价、技术等提示词模板
├─ utils/                      # 项目公共工具
├─ 货品资料/                    # 商品资料、知识库模板、数据处理脚本
├─ 90_客服知识库/               # 面向客服场景的标准知识库文档
├─ 项目解析文档/                # 面向公开阅读整理的项目拆解文档
├─ 验证文档/                    # 公开版整理过程中的验证与说明
└─ publish.bat                 # 发布辅助脚本
```

## 适合谁

- 想学习 AI 客服系统如何拆分模块的人
- 想参考 Prompt 路由、上下文管理、知识库组织方式的人
- 想做客服 Agent、私域问答机器人或售前辅助系统原型的人

## 使用建议

- 先把它当作“学习样板”，再决定要保留哪些模块进入你的项目
- 如果要改造成真实业务系统，请重新配置鉴权、日志、数据存储和风控策略
- 公开版中的知识库内容以模板化演示为主，不应直接当作真实业务数据源
