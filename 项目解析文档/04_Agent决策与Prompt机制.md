# 04 Agent 决策与 Prompt 机制

## 当前的路由思路

项目把回复逻辑拆成几个角色：

- `classify`：意图分类
- `price`：价格沟通
- `tech`：技术 / 使用 / 风险说明
- `default`：常规客服回复

## 公开版保留的学习重点

1. Prompt 文件与代码分离
2. 先分类再分发，而不是把所有问题都丢给同一个 Prompt
3. 根据不同场景控制回复风格
4. 在生成前做基本安全过滤

## 为什么这种结构适合学习

- 入口清楚，容易替换 Prompt
- 角色边界明确，方便扩展
- 可以继续往“知识库 + 检索 + 多平台”方向演进

## 公开版刻意不做的事

- 不把具体业务话术绑死在真实商品资产上
- 不把真实卖家策略写成公开可复用的成交脚本
