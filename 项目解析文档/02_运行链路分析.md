# 02 运行链路分析

## 公开版启动分两条路径

### 学习模式

- 读取 `.env.example` / `.env`
- 初始化日志
- 发现 `ENABLE_LIVE_PLATFORM=false`
- 只输出提示信息并退出

这条路径不会：

- 检查真实 Cookie
- 连接 WebSocket
- 请求真实平台接口

### 真实接入模式

只有在本地手动配置：

```env
ENABLE_LIVE_PLATFORM=true
```

并补齐关键配置后，才会进入真实接入流程：

1. 初始化 `XianyuReplyBot`
2. 初始化 `XianyuLive`
3. 解析 Cookie
4. 拉取 token
5. 建立长连接
6. 接收消息
7. 查询商品信息和上下文
8. 生成回复并发送

## 为什么公开版要默认关闭

因为原始工程的入口曾经会：

- 检查关键配置
- 在缺失时交互式要求输入
- 尝试把敏感配置写回 `.env`

这类行为不适合公开学习仓库，所以公开版改成了“严格读取配置 + 明确失败 + 默认不联网”。
